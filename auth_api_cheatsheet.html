Complete Testing Flow
1POST /signup — name, email, password, confirmPassword
↓ check Mailtrap inbox → copy token from link
2GET /verify-email/:token — paste token in URL
↓ email verified ✅
3POST /signin — copy accessToken from response
↓ set Authorization: Bearer <accessToken> in Postman
4GET /me — test protected route works
↓ change role to "admin" in MongoDB → re-signin
5GET /admin/users — test admin route with new token
↓ test forgot password flow
6POST /forgot-password → check Mailtrap → POST /reset-password/:token
↓ token expired? wait 15min → re-signin
7POST /refresh-token — get new accessToken (no body needed, uses cookie)
Common Errors & Fixes
422Validation failedCheck all required fields + password rules
401No token providedAdd Authorization: Bearer <token> header
401Token expiredHit /refresh-token → use new accessToken
403Email not verifiedCheck Mailtrap → click verify link
403Requires role: adminChange role in MongoDB → re-signin
400Invalid/expired tokenGet fresh token from email (resend if needed)
409Email already existsUse different email or login instead
423Account lockedWait 30 min or reset password
500Server errorCheck .env values + MongoDB connection
Password Rules
Min 8 characters
At least one uppercase (A-Z)
At least one lowercase (a-z)
At least one number (0-9)
Special char required
@ $ ! % * ? & # ^
Example valid password
Sachin@123 ✅
Security Features
Refresh Token Rotation
Every refresh = new token pair. Reuse = auto logout.
Account Lockout
5 wrong attempts → locked 30 min automatically.
Email Enumeration Safe
Forgot password always returns same message.
Password Change = Logout All
All devices logged out when password changes.
httpOnly Cookie
refreshToken JS se access nahi hota (XSS safe).
Hashed Tokens in DB
Verify/reset tokens SHA-256 hashed before save.
